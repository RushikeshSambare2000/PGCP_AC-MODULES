Constructor
------------

--> It is a special member function of the class which is used to initialize the object at the time of creation.

--> Its name is similar to the class name.

--> Constructor does not have any return type.

--> If we don't define any constructor, JVM internally uses default constructor.

--> There can be multiple overloaded constructors within the class. All the constructor must differ in arguments means either number of arguments should differ or order or type of arguments. This is also known as constructor overloading.

Variable Shadowing
-------------------
When local variable and instance variable have the same name then variable shadowing occurs. Means local variable hides the instance variable.

We will  use this keyword to differentiate between local and instance variable.

class Student
{
	int roll;
	
	void updateRoll(int roll)
	{
		this.roll = roll;
	}
}

---------------------------------------------
--> calling one constructor from another constructor.

--> The constructor calling using this() must be the very first statement inside the calling constructor.

Method Overloading
---------------------
It is defining one than more method with the same name but different arguments (type or order or number of arguments will vary) within the same class.
Defining multiple constructor inside a class is an example of method overloading.

class Calculator
{
	void add(int num1, int num2)
	{
		int res = num1 + num2;
		System.out.println(res);
	}
	
	void add(double num1, double num2)
	{
		double res = num1 + num2;
		System.out.println(res);
	}
	
}

class MethodOverloadingDemo
{
	public static void main(String args[])
	{
		Calculator cal = new Calculator();
		cal.add(5,3);
		cal.add(15.3,12.5);
		//cal.add('a','b');
	}
}

Type of variables inside class
-------------------------------
Instance variable ==> Variables or data which are part of the object.
Local variable ==> Variable declared inside method or block.
Static variable ==> Not a part of any object and it belongs to the entire class.

static variable
----------------
It is also known as class variable.
It is not the part of the object.
It can be accessed using class name.
For static data member memory is allocated only once.

static methods
----------------
It is also known as class methods.
It can be called using class name.
No object is required to call the static method.


--> To access any static member => className.staticMember
--> static member (static data and static methods)

Que: Can we use object to access/update the static member
Ans: YES, we can do that. But standard practice is accessing static members using the class name.


class Student
{
	int roll;
	double marks;
	
	static String noticeBoard = "Welcome Students";
	
	static void displayNoticeBoard()
	{
		System.out.println(noticeBoard);
	}
}

class StaticDemo
{
	public static void main(String args[])
	{
		System.out.println(Student.noticeBoard);
		Student.noticeBoard = "Today is holiday";
		//System.out.println(Student.noticeBoard);
		Student.displayNoticeBoard();
		
		Student s1 = new Student();	
		s1.noticeBoard = "Hey, I passed the GATE exam -- Asif"; //object.staticData
		
		Student s2 = new Student();
		s2.displayNoticeBoard();		//object.staticMethod();
	}
}

About defining constructor
----------------------------

If we define any constructor inside the class then it is our responsibility to define the zero-argument constructor also. 

If we don't define any constructor then JVM uses default constructor but if we define any constructor inside our class then JVM says that it is your responsibility to define the zero-argument constructor also.

It is a good practice to define a zero-arg constructor, if we are defining any non-zero argumnet constructor in our class but it is not mandatory.

Static and Non-static Block
----------------------------

class First
{
	
	{
		//non-static block
	}
	
	static
	{
		//static block
	}
	
}

If we use static keyword with the block then it becomes static block otherwise it is non-static block.

Non-static block is also known as initializer block.

When we create any object

Order of value initialization/calling

Step 1: Initial value at the time of declaring instance variable
Step 2: Non-static block
Step 3: Constructor

class First
{
	int a = 5;			//Step 1
	
	{
		//non-static block
		a = 10			//Step 2
	}
	
	static
	{
		//static block
	}
	
	First(int a)			//Step 3
	{
		this.a = a;
	}
	
}

Just like static method, we can not access any non-static member (data or methods) within the static block.
We can access only static member inside the static block.

If we have to perform some operation before the constructor execution we can use static and non-static block.

Static block are always called before non-static block independent of their order in the class.

Static block is executed only once.
Non-static block will be executed everytime whenever any object is created.
Static block will be executed before non-static.
We can define more than one static and non-static blocks.
Order of execution of static/non-static blocks will be same as their order of definition inside the class (top to bottom).

static blocks --> non-static blocks --> constructor

class First
{
	int a = 1;
	static int b;
	
	{
		//non-static block
		a = 2;			//OK
		System.out.println("Non-static block-1");
	}
	
	static
	{
		//static block
		//a = 3;			//ERROR; Can't access non-static member
		b = 1;				//OK; Can access only static member
		System.out.println("Static block-1");
	}
	
	{
		//non-static block
		a = 3;			//OK
		System.out.println("Non-static block-2");
	}
	
	static
	{
		System.out.println("Static block-2");
	}
	
	First(int a)
	{
		this.a = a;
	}
	
	void display()
	{
		System.out.println(this.a);
	}
	
	static void print()
	{
		System.out.println(b);
	}	
}

class StaticNonStaticBlockDemo
{
	public static void main(String args[])
	{
		First f = new First(5);			//One-arg constructor
		f.display();
		First.print();
		f = new First(10);			//One-arg constructor
		f.display();
		First.print();
	}
}

Call By Value and Call By Reference
-------------------------------------

Call by value: All the primitive data (byte,short,int,long,float,double,char,boolean) are passed by value.
Call by reference: All the objects are passed by reference.

class First
{
	int a;
	
	First(int a)
	{
		this.a = a;
	}
	
	void print()
	{
		System.out.println(a);
	}
	
	void oneFun(int x)
	{
		x = 10;
		System.out.println("call by value where x = " + x);
	}
	
}

class CallByValueAndReferenceDemo
{
	static void anotherFun(First fst)
	{
		fst.a = 15;
	}
	
	public static void main(String args[])
	{
		First f = new First(10);
		
		int p = 5;
		f.oneFun(p);		//call by value
		System.out.println(p);
		
		
		f.print();
		anotherFun(f);	//Call by reference: No new object is created here. Only reference of the 
				//Same object is passed. So if any change is made in the
				//object inside called method that will reflect here also
		f.print();
	}
}

OUTPUT:
call by value where x = 10
5
10
15

Package
==========
Package is a collection of related classes and interfaces.

Example:
java.util  ==> It contains utility classes ex. Scanner
java.sql   ==> It contains classes related to database operations
java.lang  ==> Basic package of the java language.
java.io	   ==> It contains classes for input output operations.

import java.util.*;

class First
class Second

mypack2

UsePackageDemo
---------------
import mypack.mypack1.First; 	//It will import only the First class defined inside mypack1 package.
import mypack.mypack1.*;	//It will import all the classes defined inside mypack1 package.
import mypack.mypack1.*;	//will import only classes inside mypack1 
				//and not classes under subpackages


Example code:

----------------------
package mypack.mypack1;

public class First
{
	public void funOfFirst()
	{
		System.out.println("funOfFirst");
	}
}


-----------------

public class Third

	}

-----------------

package mypack.mypack1.subpack1;

public class Fourth
{
	public void funOfFourth()
	{
		System.out.println("funOfFourth");
	}

-------------------

import mypack.mypack1.*;		//will import only classes and not classes under subpackages
import mypack.mypack2.Third;
import mypack.mypack1.subpack1.Fourth;

class UsePackageDemo
{
	public static void main(String args[])
	{
		First f = new First();

		Second s = new Second();
		s.funOfSecond();
		
		Third t = new Third();
		t.funOfThird();
		
		Fourth fr = new Fourth();
		fr.funOfFourth();
	}
}
